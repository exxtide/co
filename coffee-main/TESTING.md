## Тестирование функционала

### 1. Проверка русских статусов в истории заказов клиента

1. Войти в систему как обычный пользователь
2. Перейти в личный кабинет (иконка профиля)
3. Прокрутить до раздела "История заказов"
4. **Ожидаемый результат:** Статусы заказов отображаются на русском языке (Новый, Подтверждён, Готовится, и т.д.)

### 2. Проверка удаления категорий и подкатегорий

1. Войти в систему как администратор (is_staff = True)
2. Открыть админ-панель
3. Перейти на вкладку "Категории"
4. Нажать кнопку "Удалить" рядом с любой категорией
5. **Ожидаемый результат:** Появляется подтверждение, после подтверждения категория удаляется
6. Нажать кнопку "Удалить" рядом с любой подкатегорией
7. **Ожидаемый результат:** Появляется подтверждение, после подтверждения подкатегория удаляется

### 3. Проверка истории заказов для админа

1. Войти в систему как администратор
2. Открыть админ-панель
3. Перейти на вкладку "История заказов"
4. Выбрать дату в календаре
5. Нажать кнопку "Загрузить"
6. **Ожидаемый результат:** Отображаются только завершенные и отмененные заказы за выбранную дату
7. Оставить поле даты пустым и нажать "Загрузить"
8. **Ожидаемый результат:** Отображаются все завершенные и отмененные заказы

### 4. Проверка кнопки "Завершить" в текущих заказах

1. Войти в систему как администратор
2. Открыть админ-панель
3. Перейти на вкладку "Текущие заказы"
4. **Ожидаемый результат:** Отображаются только незавершенные заказы (не completed и не cancelled)
5. Нажать кнопку "Завершить" для любого заказа
6. Подтвердить действие
7. **Ожидаемый результат:** 
   - Заказ исчезает из списка текущих заказов
   - Появляется уведомление "Заказ завершён"
8. Перейти на вкладку "История заказов"
9. Загрузить заказы за сегодняшнюю дату
10. **Ожидаемый результат:** Завершенный заказ появляется в истории со статусом "Завершён"

## Возможные проблемы и решения

### Backend не запускается

- Проверьте, что все зависимости установлены: `pip install -r requirements.txt`
- Проверьте, что база данных настроена правильно
- Проверьте, что порт 8000 не занят другим процессом

### Frontend не запускается

- Проверьте, что Node.js установлен (версия 16+)
- Удалите `node_modules` и `package-lock.json`, затем выполните `npm install` заново
- Проверьте файл `.env` - должен быть настроен `VITE_API_ORIGIN`

### Ошибка 403 при попытке удалить категорию

- Убедитесь, что пользователь имеет права администратора (is_staff = True)
- Проверьте, что токен авторизации действителен

### История заказов пуста

- Убедитесь, что в базе данных есть заказы со статусом "completed" или "cancelled"
- Проверьте, что выбрана правильная дата
- Попробуйте загрузить без указания даты

## Создание тестовых данных

### Создать администратора

```bash
cd coffee-main/kofeinya_backend
python manage.py createsuperuser
```

### Создать тестовый заказ через Django shell

```bash
python manage.py shell
```

```python
from menu.models import Order, OrderItem, Product
from django.contrib.auth.models import User

# Получить или создать пользователя
user = User.objects.first()

# Получить продукт
product = Product.objects.first()

# Создать заказ
order = Order.objects.create(
    user=user,
    total_price=500,
    status='new',
    order_type='delivery',
    delivery_address='Тестовый адрес',
    customer_name='Тестовый клиент',
    customer_phone='+7 999 999 99 99',
    payment_method='cash'
)

# Добавить товар в заказ
OrderItem.objects.create(
    order=order,
    product=product,
    quantity=1,
    price=product.price
)

print(f"Создан заказ #{order.id}")
```

### Завершить заказ для тестирования истории

```python
order = Order.objects.get(id=1)  # Замените на нужный ID
```
