# Изменения в проекте

## Выполненные задачи

### 1. Перевод статусов заказов на русский язык в истории заказов клиента

**Файл:** `ponyatnaya-eda-main/src/pages/AccountPage.tsx`

- Добавлена функция `getStatusLabel()` для перевода статусов заказов с английского на русский
- Статусы теперь отображаются на русском языке:
  - `new` → "Новый"
  - `confirmed` → "Подтверждён"
  - `preparing` → "Готовится"
  - `delivering` → "Доставляется"
  - `completed` → "Завершён"
  - `cancelled` → "Отменён"

### 2. Добавлена возможность удаления категорий и подкатегорий

**Файлы:**
- `ponyatnaya-eda-main/src/services/api.ts` - добавлены функции `adminDeleteCategory()` и `adminDeleteSubcategory()`
- `ponyatnaya-eda-main/src/components/AdminPanel.tsx` - добавлены кнопки "Удалить" для категорий и подкатегорий

**Функционал:**
- В разделе "Категории" админ-панели теперь есть кнопка "Удалить" для каждой категории
- Для каждой подкатегории также добавлена кнопка "Удалить"
- Перед удалением выводится подтверждение

### 3. Добавлена история заказов для админ-панели

**Файлы:**
- `kofeinya_backend/menu/views.py` - добавлен endpoint `/api/orders/history/` с фильтром по дате
- `ponyatnaya-eda-main/src/services/api.ts` - добавлена функция `adminGetOrderHistory()`
- `ponyatnaya-eda-main/src/components/AdminPanel.tsx` - добавлена вкладка "История заказов"

**Функционал:**
- Новая вкладка "История заказов" в админ-панели
- Возможность выбрать дату и загрузить заказы за эту дату
- Отображаются только завершенные (`completed`) и отмененные (`cancelled`) заказы
- Показывается полная информация о заказе: клиент, адрес, состав, сумма, статус

### 4. Добавлена кнопка "Завершить" для текущих заказов

**Файлы:**
- `ponyatnaya-eda-main/src/services/api.ts` - добавлена функция `adminCompleteOrder()`
- `ponyatnaya-eda-main/src/components/AdminPanel.tsx` - добавлена кнопка "Завершить" и логика фильтрации

**Функционал:**
- Вкладка "Заказы" переименована в "Текущие заказы"
- В текущих заказах отображаются только незавершенные заказы (статус не `completed` и не `cancelled`)
- Для каждого заказа добавлена кнопка "Завершить"
- После нажатия кнопки заказ получает статус `completed` и исчезает из текущих заказов
- Завершенный заказ можно найти в "Истории заказов"

## Технические детали

### Backend (Django)

**Новый endpoint:** `GET /api/orders/history/`
- Параметры: `date` (опционально, формат YYYY-MM-DD)
- Доступ: только для staff пользователей
- Возвращает список завершенных и отмененных заказов

### Frontend (React)

**Новые компоненты и функции:**
- `loadOrderHistory()` - загрузка истории заказов
- `completeOrder()` - завершение заказа
- `deleteCategory()` - удаление категории
- `deleteSubcategory()` - удаление подкатегории
- Вкладка "История заказов" с фильтром по дате

## Использование

### Для администратора:

1. **Текущие заказы:**
   - Открыть админ-панель
   - Перейти на вкладку "Текущие заказы"
   - Изменить статус заказа через выпадающий список
   - Нажать "Завершить" для завершения заказа

2. **История заказов:**
   - Открыть админ-панель
   - Перейти на вкладку "История заказов"
   - Выбрать дату (опционально)
   - Нажать "Загрузить"

3. **Управление категориями:**
   - Открыть админ-панель
   - Перейти на вкладку "Категории"
   - Нажать "Удалить" рядом с категорией или подкатегорией
   - Подтвердить удаление

### Для клиента:

- В личном кабинете в разделе "История заказов" статусы теперь отображаются на русском языке
